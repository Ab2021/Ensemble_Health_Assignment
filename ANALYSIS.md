### 2.5 Why LR Outperforms Tree Models

The synthetic data is generated by additive, independent rules -- each gap feature adds fixed risk. This matches LR's linear modeling assumption exactly. Tree models overfit on spurious high-order interactions at this sample size (2,122 training rows). This is consistent with the master plan prediction of LR Test AUC ~ 0.699 and Capture@25% ~ 46.4%.

## 4. GenAI Explanation Engine

### 4.1 Architecture

```
Claim Data + Risk Factors -> ExplanationRequest (Pydantic)
    -> build_explanation_prompt() -> JSON-format prompt
    -> ollama.chat('gemma4:31b-cloud') -> raw JSON response
    -> Strip markdown ```json fences
    -> json.loads() -> ExplanationResponse (Pydantic validation)
    -> to_plain_text() -> clean explanation text
    
    [Parallel] -> LLMAuditLogger captures:
                 - prompt_tokens, completion_tokens, total_tokens
                 - total_duration_ms, eval_duration_ms
                 - json_parse_success, pydantic_validation_success
                 - has_disclaimer, has_action, min_length_ok
                 - quality_warnings (hallucination markers)
```

### 4.2 Pydantic Validation

**Input (ExplanationRequest):**
- `claim_id`: non-empty string
- `denial_probability`: float in [0, 1]
- `risk_estimate_label`: regex `^(High|Medium|Low)$`
- `top_risk_factors`: 0-5 items, each min_length>=1

**Output (ExplanationResponse):**
- `disclaimer`: min_length>=10, must contain "estimate" / "statistical" / "not guaranteed"
- `risk_description`: min_length>=5
- `recommended_action`: min_length>=5

**Fallback chain:** JSON parse -> strip markdown -> regex extraction -> deterministic template

## 8. Limitations

1. **Synthetic data** -- no ICD/CPT codes, clinical narratives, or payer-specific rules
2. **Temporal extrapolation** -- 2025 current claims after 2024 training; test shows 26% vs 21% denial drift
3. **Linear model** -- cannot capture complex non-linear payer-specific rules
4. **No feedback loop** -- without production outcomes, model cannot self-improve
5. **API dependency** -- GenAI explanations require Ollama Cloud; fallback ensures offline function

---

## 9. Production Recommendations

1. **Deploy Calibrated Logistic Regression** as production model -- same capture, far better probability reliability
2. **Retain LLM audit infrastructure** -- token tracking, quality validation, and fallback logging are HIPAA-safe and production-ready
3. **Use experiment tracker** for every model update -- versioned artifacts enable rollback
4. Retrain on real claims with ICD/CPT codes and line-item detail
5. Implement A/B testing: model-assisted vs manual triage (3-6 months)
6. Add dollar-weighted metrics (`expected_payment` capture)
7. Establish drift monitoring (KL divergence, denial rate shifts) with monthly reviews
8. Upgrade GenAI to production LLM with HIPAA-compliant infrastructure
9. Add active learning feedback loop from claim outcomes
